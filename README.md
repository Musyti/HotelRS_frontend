# HotelRS

Клиент-серверное приложение для управления заявками гостей отеля. Позволяет автоматизировать обработку запросов (уборка номера, технический ремонт, рум-сервис, вопросы), сократить время выполнения заявок и обеспечить разграничение прав доступа по ролям.

## 📥 Исходный код

Исходный код проекта доступен в репозитории:
- **Backend (серверная часть)**: [HotelRS_Backend](https://github.com/BelikDan0/HotelRS_backend) — Ktor сервер на Kotlin
- **Frontend (клиентская часть)**: [shared](https://github.com/Musyti/HotelRS_frontend/tree/main/shared) — Android приложение на Compose Multiplatform

## 🚀 Установка и запуск 
### Android
1. Скачайте [APK-файл](https://github.com/Musyti/HotelRS_frontend/releases/download/v1.0/androidApp-debug.apk) на Android-устройство
2. Разрешите установку из неизвестных источников (Настройки → Безопасность → Неизвестные источники)
3. Откройте скачанный файл и подтвердите установку
4. Запустите приложение через значок на рабочем столе
   
### Windows
1. Скачайте [установочный файл](https://github.com/Musyti/HotelRS_frontend/releases/download/v1.0/HotelRS-1.0.0.msi) на компьютер с Windows
2. Подтвердите безопасность файла в браузере (Вирусов нет!)
3. Откройте скачанный файл, укажите пусть установки и подтвердите ее
4. Запустите приложение через .exe файл в указанной папке при установке.

## 🔑 Начало работы

При первом запуске сервера автоматически создаётся учётная запись администратора:

| Роль | Логин | Пароль |
|------|-------|--------|
| Администратор | `admin` | `admin` |

**Для гостя** — вход по номеру телефона, указанному при заселении.

**Для сотрудника** — вход по логину и паролю, выданным администратором.

## 👥 Роли пользователей

### 👑 Администратор (ADMIN)
- Просмотр всех заявок
- Фильтрация заявок по статусу (Новые / В работе / Выполнены)
- Изменение статуса любой заявки
- Удаление заявок
- Управление пользователями (гости и сотрудники)
- Просмотр деталей заявки по нажатию

### 👔 Сотрудник (STAFF)
- Просмотр заявок категорий «Рум-сервис» и «Вопросы и пожелания»
- Фильтрация заявок по статусу
- Изменение статуса заявки (Новая → В работе → Выполнена)

### 🧹 Горничная (CLEANER)
- Просмотр заявок категории «Уборка номера»
- Изменение статуса заявки

### 🔧 Мастер (MASTER)
- Просмотр заявок категории «Технический ремонт»
- Изменение статуса заявки

### 🏨 Гость (GUEST)
- Создание новых заявок с выбором категории
- Просмотр своих заявок
- Отслеживание статуса выполнения

## 📱 Функционал приложения

### Экран входа
- Выбор роли (Гость / Сотрудник)
- Ввод номера телефона (для гостя) или логина/пароля (для сотрудника)
- Валидация номера телефона (10–15 цифр, может начинаться с +)

### Главный экран (гость)
- Список созданных заявок
- Кнопка «Создать» для новой заявки
- Кнопка «Обновить» для обновления списка
- Карточки заявок с категорией, статусом, описанием и датой

### Экран создания заявки
- Выбор категории из выпадающего списка
- Поле для описания проблемы
- Отправка заявки на сервер

### Панель сотрудника / админа
- Вкладка «Заявки» — список заявок с фильтром по статусу
- Кнопки быстрого изменения статуса (Новая / В работе / Выполнена)
- Кнопка удаления заявки (только для администратора)
- По нажатию на карточку — просмотр деталей заявки

### Панель администратора
- Вкладка «Пользователи»
- Просмотр списка гостей и сотрудников
- Редактирование информации о пользователе
- Удаление пользователей (кроме администратора)

## 🔄 Взаимодействие с сервером

Приложение взаимодействует с сервером через REST API:

| Метод | Эндпоинт | Назначение |
|-------|----------|------------|
| POST | `/api/auth/login` | Авторизация |
| POST | `/api/tickets` | Создание заявки |
| GET | `/api/guest/tickets` | Заявки гостя |
| GET | `/api/admin/tickets` | Все заявки (админ) |
| GET | `/api/staff/tickets` | Заявки по роли |
| PUT | `/api/tickets/{id}/status` | Изменение статуса |
| DELETE | `/api/tickets/{id}` | Удаление заявки |
